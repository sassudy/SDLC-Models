Repositories, Branches and Commits

A Repository is the location where the code history is stored.
Inside the Repository we have a Branch, which is the folder that contains our Commits.
A Commit stands for the different stages and versions of our sourcecode which we save in the Branch.

$git init -Initializing a Git repository, telling Git to manage the project in this folder.

$git add . -Adding all the project files to the tracking logic of Git.

$git status -View the changes made.

$git commit -m "descriptive text here" -Telling Git to commit to the changes and add our code to the Branch, saving it.
Adding -m to give the Commit a name.

$git branch -Git automatically creates a Branch.
