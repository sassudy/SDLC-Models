Git and GitHub

Differences between Git and GitHub

Git Repositories, Branches and Commits

A Repository is the location where the code history is stored.
Inside the Repository we have a Branch, which is the folder that contains our Commits.
A Commit stands for the different stages and versions of our sourcecode which we save in the Branch.

Git does not create copies of Your commits, it tracks changes You make to the code.

$git init -Initializing a Git repository, telling Git to manage the project in this folder.

$git add . -Adding all the project files to the tracking logic of Git.

$git status -View the changes made.

$git commit -m "descriptive text here" -Telling Git to commit to the changes and add our code to the Branch, saving it.
Adding -m to give the Commit a name.

$git branch -Git automatically creates a Branch. Can see the branches we have.

$git log -View the commits created.

$git checkout --. -You can delete the last commit You made to the code.

$git checkout -b say-something -Create a new branch.

GitHub push and fetch

In GutHub you create a remote repository where your code is stored.

Create a GitHub repository, add a name for it.

Next You have to connect Your file to the repository, for this, before you have any files, GitHub also provides hints.

To begin the process you add the following lines to the terminal

$git init$git add . >$git commit -m "descriptive text here"

Next in the hints GitHub provides you will have the link to connect $git remote add origin (link)

Now You need to push your files $git push -u origin master

To push files to an already existing and connected repository add the following codes

$git add . $git commit -m "descriptive text here" $git push

If we want to access codes from GitHub, we can clone or download the repository.

We can copy the link GitHub provides and opening VS Code and a new file add the following codes

$git clone (link) Now the entire repository is cloned locally

To access the updates made in GitHub we can fetch the files $git fetch
